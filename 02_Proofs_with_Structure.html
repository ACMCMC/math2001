2. Proofs with structure

2.1. Intermediate steps

2.1.1. Example

Every proof we’ve seen so far has been a single calculation. More typically, though, a proof will have a more complex structure, with facts established at an early stage which are not used immediately, but instead brought in later.

For example, here’s the algebra problem from Example 1.3.3.

Problem

Let \(a\) and \(b\) be real numbers and suppose that \(a-5b=4\) and \(b+2=3\). Show that \(a=9\).

We previously solved it by a single long calculation,

\[\begin{split}a &= \ldots\\ &= \ldots\\ &= 9.\end{split}\]

But another way to express the solution – maybe more natural, and closer to how you might have learned to solve it in high school – is to first solve for \(b\), and then substitute that in to help solve for \(a\).

Solution

Since \(b+2=3\), we have \(b=1\). Therefore

\[\begin{split}a &= (a - 5b) + 5b\\ &= 4 + 5 \cdot 1 \\ &= 9.\end{split}\]
example {a b : ℝ} (h1 : a - 5 * b = 4) (h2 : b + 2 = 3) : a = 9 :=
begin
  have hb : b = 1,
  addarith [h2],
  calc a = (a - 5 * b) + 5 * b : by ring
  ... = 4 + 5 * 1 : by rwa [h1, hb]
  ... = 9 : by ring,
end

2.1.2. Example

Let’s redo another example, this time Example 1.4.2. The problem was:

Problem

Let \(r\) and \(s\) be rational numbers, and suppose that \(s+3\geq r\) and \(s+r \leq 3\). Show that \(r\leq 3\).

We did it before by a single, clever, calculation, but the following solution, though longer, might be easier to come up with.

Solution

From \(s + 3 \geq r\) we have \(r \leq 3 + s\), and from \(s + r \leq 3\) we have \(r \leq 3 - s\). Therefore

\[\begin{split}r&=\frac{r+r}{2}\\ &\leq \frac{(3+s)+(3-s)}{2}\\ &=3.\end{split}\]

There are two intermediate steps in this problem: proving that \(r \leq 3 + s\) and that \(r \leq 3 - s\).

example {r s : ℚ} (h1 : s + 3 ≥ r) (h2 : s + r ≤ 3) :
  r ≤ 3 :=
begin
  have h3 : r ≤ 3 + s,
  sorry,
  have h4 : r ≤ 3 - s,
  sorry,
  calc r = (r + r) / 2 : by sorry
  ... ≤ ((3 - s) + (3 + s)) / 2 : by sorry
  ... = 3 : by sorry
end

2.1.3. Example

Problem

Let \(a\) and \(b\) be real numbers and suppose that \(-b \le a \le b\). Show that \(a ^ 2 \le b ^ 2\).

Solution

By the first part of the hypothesis, \(0 \le b + a\), and by the second part of the hypothesis, \(0 \le b - a\).

Therefore \((b + a)(b - a)\) is nonnegative, so

\[\begin{split}a ^ 2 & &\le a ^ 2 + (b+a)(b-a)\\ &= b ^ 2.\end{split}\]
example (a b : ℝ) (h1 : -b ≤ a) (h2 : a ≤ b) : a ^ 2 ≤ b ^ 2 :=
begin
  sorry
end

2.1.4. Example

Problem

Let \(a\) and \(b\) be real numbers and suppose that \(a \le b\). Show that \(a ^ 3 \le b ^ 3\).

Notice that we are not assuming that \(a\) and \(b\) be positive, so our easier tricks for the behaviour of inequalities under multiplication/powers do not apply.

Solution

By the hypothesis, \(0 \le b - a\).

Therefore \(\frac{(b - a)[(b - a)^2+3(b+a)^2]}{4}\) is nonnegative, so

\[\begin{split}a ^ 3 &\le a ^ 3 + \frac{(b - a)[(b - a)^2+3(b+a)^2]}{4}\\ &= b ^ 3.\end{split}\]
example (a b : ℝ) (h : a ≤ b) : a ^ 3 ≤ b ^ 3 :=
begin
  sorry
end

2.2. Invoking lemmas

2.2.1. Example

Here’s a style of problem we haven’t seen before, in which the goal is a disequality, \(x\ne 1\), rather than an equality (\(=\)) or inequality (\(\le\), \(<\), \(\ge\), \(>\)).

Problem

Let \(x\) be a rational number, and suppose that \(3x=2\). Show that \(x\ne 1\).

Solution

It suffices to prove that \(x< 1\). Indeed,

\[\begin{split}x &= (3x)/3 \\ &=2/3 \\ &< 1.\end{split}\]

What happened in this problem? We used a piece of “general knowledge”, that if one number is strictly smaller than another then they can’t be the same. Or at least, this feels like general knowledge – just common sense! – but really it is a lemma: a fact that has been proved already 1 by us or someone else, and which we are allowed to invoke in our proof.

When working in Lean, if we want to invoke a lemma in this way, we need to call it out by name. Someone earlier, in the big Lean mathematical library, has proved this fact and named it ne_of_lt: 2

lemma ne_of_lt {a b : ℚ} (h : a < b) : a ≠ b :=

We can invoke this lemma in our problem using the Lean apply tactic. At the start of our work on the problem,

example {x : ℚ} (hx : 3 * x = 2) : x ≠ 1 :=
begin

end

our goal state looks like this:

x : ℚ
hx : 3 * x = 2
⊢ x ≠ 1

When we apply the lemma, with our cursor at the end of this line,

example {x : ℚ} (hx : 3 * x = 2) : x ≠ 1 :=
begin
  apply ne_of_lt,

end

the goal state has changed to this:

x : ℚ
hx : 3 * x = 2
⊢ x < 1

So apply ne_of_lt changes (only) the goal: it transforms the goal x ≠ 1 into the goal x < 1, which we can then solve by our usual methods for inequalities.

example {x : ℚ} (hx : 3 * x = 2) : x ≠ 1 :=
begin
  apply ne_of_lt,
  calc x = (3 * x) / 3 : by ring
  ... = 2 / 3 : by rw hx
  ... < 1 : by norm_num,
end

Comparing the text proof and the Lean proof, you will notice that the phrasing in invoking the lemma is rather different. In text, I said,

It suffices to prove that \(x< 1\). Indeed, ….

That is, I effectively stated what the new goal would be, and left the reader to figure out what piece of general knowledge I used to make this change to the goal. In Lean, I don’t need to state what the new goal is, because my reader can find this out for herself by inspecting the goal state. But I do need to explicitly mention the name of the lemma,

apply ne_of_lt,

because “it’s general knowledge!” is not a precise enough justification for Lean to find it.

Problem

Let \(y\) be a real number. Show that \(y ^ 2 + 1\ne 0\).

Solution

It suffices to prove that \(0 < y ^ 2 + 1\), which is clear since squares are nonnegative.

2.2.3. Example

Problem

Let \(a\) and \(b\) be real numbers, and suppose that \(a^2+b^2=0\). Show that \(a ^ 2 = 0\).

Solution

It suffices to prove that both \(a ^ 2 \le 0\) and \(a ^ 2 ≥ 0\). The second is clear since squares are nonnegative. For the first,

\[\begin{split}a ^ 2 &= a ^ 2 + 0 \\ &\le a ^ 2 + b ^ 2\\ &=0.\end{split}\]

In Lean use this lemma, the “antisymmetry of the \(\le\) relation”:

lemma le_antisymm {a b : α} (h1 : a ≤ b) (h2 : b ≤ a) : a = b :=

end

2.2.4. Example

Here is another commonly-used lemma which replaces one goal by two (hopefully easier) goals: if you know that \(0<ab\) and \(0 \le a\), then you can conclude that \(0<b\).

(You might think of this as encapsulating the fact that the quotient of a positive number by a nonnegative number is positive. But working with division in proofs is always painful – it brings along the need for constant checks that denominators are nonzero – so it’s convenient to have a formulation like this which doesn’t mention division.)

Let’s do a problem whose solution has several steps: we use this lemma to prove an intermediate goal, and then use the intermediate goal in the conclusion.

Problem

Let \(a\) and \(b\) be rational numbers, and suppose that \(ab=1\) and \(1 \le a\). Show that \(1 \ge b\).

Solution

We first show that \(b\) is positive. This is true since \(ab=1\) is positive and \(a\) is nonnegative.

Now, since \(b\) is positive, the following calculation is valid:

\[\begin{split}1 &= ab \\ &\ge 1 \cdot b\\ &=b.\end{split}\]

Here’s that lemma (i.e., the lemma that you can show a number is positive by showing that its product with some other nonnegative number is positive) in Lean.

lemma pos_of_mul_pos_right' {y : ℚ} (x : ℚ) (h : 0 < x * y) (hx : 0 ≤ x) : 0 < y :=

When you’re using this lemma, you’ll need to specify a piece of the plan that Lean can’t figure out: the number which plays the x role in the statement, i.e. the number we’ll be multiplying by. You can provide this information to Lean by filling in that number when you apply the lemma. That is, use a term like pos_of_mul_pos_right' 23 or pos_of_mul_pos_right' q or pos_of_mul_pos_right' (m ^ 2 + 1), rather than just pos_of_mul_pos_right' on its own.

2.2.5. Example

Finally, here’s an example in which we need to use several lemmas. Try to pick them out in the text proof as they are used.

Problem

Let \(a\) and \(b\) be real numbers, and suppose that \(ab=0\) and \(0 < a\). Show that \(b = 0\).

Solution

It suffices to prove that both \(b \le 0\) and \(b ≥ 0\).

The first is true since \(ab=0\) is nonpositive and \(a\) is positive.

The second is true since \(ab=0\) is nonnegative and \(a\) is positive.

In Lean, the relevant lemmas are the antisymmetry lemma, together with two variants on the lemma deducing positivity from positivity of a multiple.

lemma eq_zero_of_mul_pos_eq_zero {b : ℝ} (a : ℝ) (h : a * b = 0) (ha : 0 < a) :
  b = 0 :=
begin
  sorry
end

Unusually, we’re giving this problem a name in Lean, eq_zero_of_mul_pos_eq_zero, for future reference. This is because we’ll use it later in Example 2.3.2.

2.3. “Or” and proof by cases

2.3.1. Example

Problem

Let \(n\) be any natural number. Show that \(n ^ 2 \ne 2\).

Solution

We consider separately the cases \(n \le 1\) and \(2 \le n\).

Case 1 (\(n \le 1\)): It suffices to prove that \(n ^ 2 < 2\). Indeed,

\[\begin{split}n ^ 2 & \le 1 ^ 2\\ &<2.\end{split}\]

Case 2 (\(2 \le n\)): It suffices to prove that \(n ^ 2 > 2\). Indeed,

\[\begin{split}2 &< 2 ^ 2\\ & \le n ^ 2.\end{split}\]
example {n : ℕ} : n ^ 2 ≠ 2 :=
begin
  have hn : n ≤ 1 ∨ 2 ≤ n := le_or_lt n 1,
  cases hn with hn hn,
  apply ne_of_lt,
  have : 0 ≤ n := sorry, -- soon will be built into `positivity`
  calc n ^ 2 ≤ 1 ^ 2 : by ineq_tac [hn]
  ... < 2 : by norm_num,
  apply ne_of_gt,
  calc 2 < 2 ^ 2 : by norm_num
  ... ≤ n ^ 2 : by ineq_tac [hn],
end

2.3.2. Example

We showed in Example 2.2.5 that if \(ab=0\) and \(0 < a\), then \(b = 0\). Later, in the exercises to that section, we showed that that if \(ab=0\) and \(a < 0\), then \(b = 0\).

Using a case analysis on \(a\), we can now prove a more general lemma valid for all values of \(a\).

Problem

Let \(a\) and \(b\) be real numbers, and suppose that \(ab = 0\). Show that either \(a = 0\) or \(b = 0\).

Solution

We consider separately the three cases \(a < 0\), \(a=0\) and \(0 < a\).

Case 1 (\(a < 0\)): We have that \(ab=0\) and \(a<0\). So by the lemma proved in a previous exercise, we have that \(b=0\), the second alternative.

Case 2 (\(a = 0\)): This is itself the first alternative in our goal.

Case 3 (\(0 < a\)): We have that \(ab=0\) and \(0 <a\). So by the lemma proved in Example 2.2.5, we have that \(b=0\), the second alternative.

example {a b : ℝ} (h : a * b = 0) : a = 0 ∨ b = 0 :=
begin
  have ha : a < 0 ∨ a = 0 ∨ 0 < a := lt_trichotomy a 0,
  cases ha with ha ha,
  right,
  apply eq_zero_of_mul_neg_eq_zero a,
  apply h,
  apply ha,
  cases ha with ha ha,
  left,
  apply ha,
  right,
  apply eq_zero_of_mul_pos_eq_zero a,
  apply h,
  apply ha,
end

The problem we just solved actually already appears in the Lean mathematical library under the name eq_zero_or_eq_zero_of_mul_eq_zero. It’s clearly an important fact, and we’ll use it and its variants in later problems.

2.3.3. Example

In Example 2.3.1 we showed that no natural number squares to 2. It is also true that no integer squares to 2, but since order laws are more complicated when negative numbers are involved, the proof is more complicated, requiring cases within cases.

Problem

Let \(n\) be any integer. Show that \(n ^ 2 \ne 2\).

Solution

We consider separately the cases \(n \le 0\) and \(1 \le n\).

Case 1 (\(n \le 0\)): In this case we have \(0 \le -n\). We consider separately the cases \(-n \le 1\) and \(2 \le -n\).

Case 1(i) (\(-n \le 1\)): It suffices to prove that \(n ^ 2 < 2\). Indeed,

\[\begin{split}n ^ 2 &= (-n) ^ 2\\ & \le 1 ^ 2\\ &<2.\end{split}\]

Case 1(i) (\(2 \le -n\)): It suffices to prove that \(n ^ 2 > 2\). Indeed,

\[\begin{split}2 &= (-n) ^ 2\\ &< 2 ^ 2\\ & \le n ^ 2.\end{split}\]

Case 2 (\(1 \le n\)): We consider separately the cases \(n \le 1\) and \(2 \le n\).

Case 2(i) (\(n \le 1\)): It suffices to prove that \(n ^ 2 < 2\). Indeed,

\[\begin{split}n ^ 2 & \le 1 ^ 2\\ &<2.\end{split}\]

Case 2(ii) (\(2 \le n\)): It suffices to prove that \(n ^ 2 > 2\). Indeed,

\[\begin{split}2 &< 2 ^ 2\\ & \le n ^ 2.\end{split}\]

When a proof becomes this complicated, you may find it helpful to enclose the separate parts in braces, as follows.

example {n : ℤ} : n ^ 2 ≠ 2 :=
begin
  have hn0 : n ≤ 0 ∨ 1 ≤ n := le_or_lt n 0,
  cases hn0 with hn0 hn0,
  { have : 0 ≤ - n,
    addarith [hn0],
    have hn : -n ≤ 1 ∨ 2 ≤ -n := le_or_lt (-n) 1,
    cases hn,
    { apply ne_of_lt,
      calc n ^ 2 = (-n) ^ 2 : by ring
      ... ≤ 1 ^ 2 : by ineq_tac [hn]
      ... < 2 : by norm_num },
    { apply ne_of_gt,
      calc 2 < 2 ^ 2 : by norm_num
      ... ≤ (-n) ^ 2 : by ineq_tac [hn]
      ... = n ^ 2 : by ring } },
  { have hn : n ≤ 1 ∨ 2 ≤ n := le_or_lt n 1,
    cases hn with hn hn,
    { apply ne_of_lt,
      calc n ^ 2 ≤ 1 ^ 2 : by ineq_tac [hn]
      ... < 2 : by norm_num },
    { apply ne_of_gt,
      calc 2 < 2 ^ 2 : by norm_num
      ... ≤ n ^ 2 : by ineq_tac [hn] } },
end

2.3.4. Example

Let’s solve a quadratic equation; this is a classic situation where the result has an “or” in it. We will get to use the fact proved in Example 2.3.2: if the product of two numbers is zero, then one of them is zero.

Problem

Let \(x\) be a real number for which \(x^2-3x+2=0\). Show that either \(x=1\) or \(x=2\).

Solution

We have that

\[\begin{split}(x-1)(x-2) &= x ^ 2 - 3x+2\\ &=0.\end{split}\]

So either \(x-1=0\) or \(x-2=0\).

If \(x-1=0\), then \(x=1\).

If \(x-2=0\), then \(x=2\).

In this solution, the case division argument is written a little more casually than in previous examples. The phrases “If \(x-1=0\)” and “If \(x-2=0\)” silently introduce the two cases, and it is left to the reader to observe that \(x=1\) is the left alternative of “\(x=1\) or \(x=2\)”, which finishes the problem (similarly for the second case).

Solution

We have that

So either \(a=0\) or \(a=0\), and in either case we are done.
